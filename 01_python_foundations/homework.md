## Assignment 1: Environment checklist

1. Write down your **Python version** (from `python --version` or `python3 --version`).
2. Confirm you can run **one** `.py` file from the terminal and paste the command you used plus the program output.

**Deliverable:** Short text or screenshot showing version + successful run.

## Assignment 2: Personal intro program

Write a script `intro.py` that:

1. Asks the user for their **first name** and **favorite color** (use `input()`).
2. Prints a friendly one-line message that uses both values (use `print` with commas, or string concatenation with `+`).

**Deliverable:** `intro.py` + example run with sample input/output.

---

## Assignment 3: Types practice

Create `types_practice.py` that:

1. Creates variables: an integer, a float, a string, and a boolean (use clear names).
2. Prints each variable and its type using `type(...)`.
3. Prints **one** extra line that uses your string and boolean in a short message (for example: “Is it sunny?” plus your `True` or `False`), using `print` with commas—no formulas, just showing those values.

**Deliverable:** `types_practice.py` + pasted output.

---

## Assignment 4: Pseudocode then code

1. Write **pseudocode only** (plain steps, not Python) for a tiny program that: stores **two** pieces of text you choose (for example a name and a city, or a pet’s name and a color), then **prints two lines** that use those values.
2. Implement the same steps in Python in `pseudocode_practice.py` using variables and `print`.

**Deliverable:** pseudocode in the same file as comments at the top **or** in a short `pseudocode.txt`, plus `pseudocode_practice.py` + output.

## Assignment 5: Short reflection (optional)

In 5–8 sentences: What was easiest and what was hardest in Section 1? Name **one** error you saw while debugging and how you fixed it (or how you would fix it next time). What will you review before Section 2?
