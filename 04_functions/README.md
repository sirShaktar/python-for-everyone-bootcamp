# Section 4: Functions

This section teaches how to **name and reuse** behavior with **functions**, how arguments and return values work, flexible signatures with `*args` and `**kwargs`, and **scope**.

## Learning goals

By the end of this section, you will be able to:

- Define functions with `def` and call them with arguments.
- Distinguish parameters (definition) from arguments (call site).
- Return values with `return` and handle “no useful return” (`None`).
- Use default parameters safely and understand evaluation timing pitfalls at a high level.
- Accept variable positional and keyword arguments with `*args` and `**kwargs`.
- Explain local vs global scope and use naming patterns that avoid bugs.
